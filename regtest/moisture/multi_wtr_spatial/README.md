# Multi-Station Weather Spatial Interpolation Test

## Purpose

This regression test validates the spatial interpolation of temperature (T) and relative humidity (RH) from multiple weather stations using Inverse Distance Weighting (IDW). It exercises the new `apply_multi_wtr_TRH_to_spatial()` function and verifies that spatially-varying fuel moisture content is correctly computed from interpolated weather data.

## Test Setup

The test creates 3 weather stations at different locations with distinct T/RH conditions:

| Station | Location (UTM) | Temperature | RH | Description |
|---------|---------------|-------------|-----|-------------|
| 1 | (330000, 3775000) SW | 40°C (104°F) | 15% | Hot & Dry |
| 2 | (331000, 3775000) SE | 30°C (86°F) | 30% | Moderate |
| 3 | (330500, 3776000) N | 20°C (68°F) | 60% | Cool & Wet |

All stations have the same wind speed (10 mph from 270°) to isolate the T/RH gradient effect.

## Expected Behavior

The IDW interpolation creates a smooth T/RH gradient across the domain:

- **Southwest (near Station 1)**: Hottest/driest conditions → lowest fuel moisture → fastest fire spread
- **Southeast (near Station 2)**: Moderate conditions → moderate fuel moisture
- **North (near Station 3)**: Coolest/wettest conditions → highest fuel moisture → slowest fire spread

The fire ignites at the domain center (330500, 3775500) and should spread asymmetrically, with:
- Faster spread toward the hot/dry southwest
- Slower spread toward the cool/wet north

## Validation Checks

1. **Station Loading**: All 3 .wtr files are parsed successfully
2. **IDW Interpolation**: T and RH vary smoothly across the domain following the IDW formula
3. **Spatial EMC**: Dead-fuel moisture (M_d1, M_d10, M_d100) varies with local T/RH
4. **Fire Spread Asymmetry**: Rate of spread correlates with local moisture content

## Files

- `create_stations.py` - Python script to generate station list and .wtr files (run before test)
- `inputs.i` - Input file for the wildfire_levelset simulation
- `stations.csv` - Station list (generated by create_stations.py)
- `station1.wtr`, `station2.wtr`, `station3.wtr` - Weather data files (generated by create_stations.py)

## Running the Test

```bash

```

## Implementation Details

This test specifically exercises:

- `load_multi_wtr_stations()` in `multi_wtr_weather.H` - Loads station data
- `apply_multi_wtr_TRH_to_spatial()` in `multi_wtr_weather.H` - Spatially interpolates T/RH to MultiFabs
- `apply_solar_emc_to_spatial_moisture()` (spatial T/RH overload) in `solar_radiation.H` - Computes per-cell EMC from spatial T/RH
- `apply_solar_radiation_step()` (spatial T/RH overload) in `solar_radiation.H` - Integrates spatial weather with shade-adjusted EMC
