# Solar Topographic Horizon Shading Regression Test

## Purpose

Tests the FARSITE full topographic horizon scan introduced in
`src/solar_radiation.H`:

- **`compute_topographic_horizon_angles`** — one-time CPU precomputation that
  gathers the global elevation field via MPI AllReduce and ray-marches 8
  compass directions to find the maximum terrain-elevation (horizon) angle per
  cell.
- **Horizon-aware `compute_shade_fraction_mf`** — each timestep, the
  precomputed horizon angles are consulted in the GPU shade kernel: the cell is
  marked shaded when the solar elevation is below the interpolated horizon
  angle in the solar-azimuth direction.

Local surface-normal shading (`cos_i`) cannot detect shadows cast by terrain
dozens or hundreds of cells away.  This test exercises that extended capability
by placing an ignition on the floor of a ridge-flanked canyon.

## Terrain

Generated by `create_terrain.py` (no external dependencies):

| Feature | UTM x [m] | Elevation [m] |
|---------|-----------|---------------|
| W ridge | 330 000 | ~200 |
| Canyon floor | 330 500 | ~50 |
| E ridge | 331 000 | ~200 |

A 20 m N-S linear gradient makes slope and aspect non-trivial across the
domain.

**Nominal ridge horizon angle** from the canyon floor:

```
θ_hz ≈ atan2(200 − 50, 500) ≈ 16.7°
```

At the configured start time (06:30 AM PDT, July 1, 2024, lat 34.1° N) the
solar elevation is approximately 10°, which is **below** the ridge horizon.
The horizon scan therefore shadows the canyon floor even though the flat valley
floor would satisfy `cos_i > 0` under local surface-normal shading.

## Configuration

| Parameter | Value |
|-----------|-------|
| Domain | 1 000 m × 1 000 m, 64 × 64 cells (~15.6 m) |
| Propagation | level-set |
| Fuel | FM4 chaparral, M_f = 8 % |
| Wind | 3 m/s eastward |
| Solar start | 06:30 local (July 1, 2024, PDT) |
| `use_topographic_horizon` | **1** (key feature under test) |
| `horizon_scan_max_dist_m` | 800 m |
| Diurnal moisture | enabled (provides T_air / RH for shade-adjusted EMC) |

## Key Input Parameters

```ini
solar_radiation.enable                  = 1
solar_radiation.use_topographic_horizon = 1
solar_radiation.horizon_scan_max_dist_m = 800.0
```

Set `use_topographic_horizon = 0` to skip the MPI global-elevation gather and
O(N²) CPU sweep — recommended on large domains or many MPI ranks where the
one-time cost is prohibitive.

## How to Run

```bash
# From the repository root (after building):
cd regtest/terrain/solar_horizon_shading
python3 create_terrain.py
../../../build/levelset inputs.i
```

Via CTest (Python setup registered as a CMake fixture):

```bash
cd build
ctest -R solar_horizon_shading --output-on-failure
```

## Expected Behaviour

1. `create_terrain.py` prints terrain stats including the nominal horizon angle.
2. The solver prints `"Topographic horizon angles precomputed for all 8 directions."`.
3. The shade-fraction field at t = 0 shows the canyon floor as fully shaded
   (`shade_fraction = 1.0`) while ridge crests are partially insolated.
4. Fire spread continues for 40 steps without errors.
